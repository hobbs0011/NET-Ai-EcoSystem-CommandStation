Severity: BLOCKING | ID: CR_ARCH_01

The Problem:

Your assets define:



5 stages (the "container")

60 modules (the "content")

ZERO specification of which modules go with which stages

This is like designing a LEGO instruction manual with a parts list but no assembly diagram.

Evidence:



// Template has stages with placeholders

// Arsenal has modules with descriptions

Your V1.3 UI cannot be built without answering: "What does the module selector show for each stage?"

Three failure modes:Show all 60 modules for every stage → DA_ARS_01 paralysis × 5 stages

Hard-code mappings in UI → Data/logic coupling, unmaintainable

"title": "The Vocabulary Forge",

"category": "vocabulary"

This allows your UI to:



Filter modules contextually per stage

Reduce choice sets from 60 → ~8-12 per stage

Provide intelligent defaults (primaryStage)

Severity: BLOCKING | ID: CR_ARCH_02
